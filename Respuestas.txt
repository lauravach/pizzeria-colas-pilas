1. ¿Por qué un sistema de delivery usa Queue para los pedidos pero Stack para la bitácora? 
¿Qué problema operativo surgiría si invertimos las estructuras?

Respuesta: El queue permite que se atiendan por orden de llegada, lo que hará que el cliente sea sienta atendido
de manera adecuada. A diferencia de los pedidos que deben ser realizados con un stack porque el ultimo que entra 
corresponde al primer cliente que llego y debe salir correspondiente al orden de llegada de los clientes.

El problema operativo sería que si se usa el stack para el sistema de delivery es que se atendería primero al ultimo
que llegó lo cual crearía una molestia para el primer cliente que llego porque se atiende a alguien que lleva esperando 
menos tiempo primero. Si por otro lado usamos queue en el área de pedidos se atendería primero al ultimo que llegó

4. ¿Qué ventaja tiene entregar mediante Fork + Pull Request 
en lugar de un archivo comprimido? ¿Cómo facilita la evaluación, la trazabilidad y la retroalimentación?

Entregar mediante Fork + Pull Request (PR) ofrece ventajas abismales sobre un archivo comprimido .zip o .rar 
porque transforma una entrega estática en un proceso dinámico porque el archivo comprimido es como una imagen
estatica, a diferencia del fork y PR que pude actualizarse de una manera tambien mas sencilla.
